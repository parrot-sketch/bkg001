## Seeding the Database

To generate all test credentials, run:

```bash
npm run db:seed
```

This will:
1. Clear all existing data
2. Create all users listed above
3. Create sample appointments, consultations, and related data
4. Assign patients to doctors
5. Create working schedules for doctors

## Important Notes

### Security
⚠️ **WARNING:** These are development/test credentials only.  
❌ **DO NOT** use these passwords in production environments.  
✅ **DO** change all passwords before deploying to production.

### Password Rules
- Passwords are hashed using bcrypt with 10 salt rounds
- Patient passwords are their phone numbers (digits only, no + or spaces)
- Staff passwords follow the pattern: `{role}123`

### Login URLs
- Admin: `/admin`
- Doctor: `/doctor`
- Nurse: `/nurse`
- Frontdesk: `/frontdesk`
- Patient: `/patient`

### Creating New Test Users
If you need additional test users, you can:
1. Modify `prisma/seed.ts` and re-run `npm run db:seed`
2. Create users manually through the admin interface
3. Use the API endpoints to create users programmatically

## Troubleshooting

### Can't log in with a patient account?
1. Verify the email is correct
2. Use the phone number as password (remove the `+` sign)
3. Check that the user was created in the database

### Forgot which email belongs to which role?
- Admin emails: `@nairobisculpt.com` with role ADMIN
- Doctor emails: `@nairobisculpt.com` with firstname.lastname format
- Nurse emails: `@nairobisculpt.com` with firstname.lastname format
- Frontdesk emails: `@nairobisculpt.com` with firstname.lastname format
- Patient emails: Various formats, can include `@patient.nairobisculpt.com` or personal emails

### Need to reset a password?
- Admin users can reset passwords through the admin interface
- Or modify `prisma/seed.ts` and re-run the seed script (⚠️ this will delete all data)
